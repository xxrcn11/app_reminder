# Spec: Apple Reminder Web App

## 1. 개요

Apple Reminder 앱의 핵심 기능을 웹 버전으로 구현합니다.

| 항목 | 내용 |
|------|------|
| Backend | Spring Boot 3.5 + MyBatis + MySQL |
| Frontend | Next.js (latest) |
| 목표 | Apple Reminder 앱과 최대한 유사한 UI/UX로 웹 버전 구현 |

---

## 2. 핵심 기능

### 2.1 리스트 관리

| 기능 | 설명 |
|------|------|
| 리스트 CRUD | 리스트 생성, 조회, 수정, 삭제 |
| 리스트 색상 | 리스트별 색상 지정 (12가지 프리셋) |
| 리스트 아이콘 | 리스트별 아이콘 선택 |
| 리스트 정렬 | 드래그 앤 드롭으로 순서 변경 |

### 2.2 리마인더(할 일) 관리

| 기능 | 설명 |
|------|------|
| 리마인더 CRUD | 리마인더 생성, 조회, 수정, 삭제 |
| 완료 처리 | 체크박스로 완료/미완료 토글 |
| 제목 & 메모 | 제목(필수) + 메모(선택) 입력 |
| 마감일 | 날짜 및 시간 설정 |
| 우선순위 | 없음, 낮음, 보통, 높음 (4단계) |
| 플래그 | 중요 표시 플래그 토글 |

### 2.3 스마트 리스트 (기본 제공)

| 스마트 리스트 | 필터 조건 |
|--------------|----------|
| 오늘 | 마감일이 오늘인 리마인더 |
| 예정 | 마감일이 설정된 미완료 리마인더 (날짜별 그룹핑) |
| 전체 | 모든 미완료 리마인더 |
| 플래그 지정됨 | 플래그가 설정된 리마인더 |
| 완료됨 | 완료 처리된 리마인더 |

### 2.4 검색

| 기능 | 설명 |
|------|------|
| 전체 검색 | 제목, 메모 기준 키워드 검색 |

## 3. UI/UX 디자인 (Apple Reminder 스타일)

### 3.1 디자인 원칙

- Apple Reminder macOS 앱의 룩앤필을 최대한 재현
- 깔끔하고 여백이 넉넉한 레이아웃
- 부드러운 애니메이션과 트랜지션
- 시스템 폰트 사용 (`-apple-system, BlinkMacSystemFont, 'Segoe UI'`)

### 3.2 색상 체계

| 용도 | 색상 |
|------|------|
| 배경 (사이드바) | `#F2F1F6` (연한 회색) |
| 배경 (메인 콘텐츠) | `#FFFFFF` |
| 텍스트 (기본) | `#1C1C1E` |
| 텍스트 (보조) | `#8E8E93` |
| 구분선 | `#E5E5EA` |
| 선택 상태 (사이드바) | `rgba(0,0,0,0.05)` 배경 + 볼드 |
| 완료 체크 애니메이션 | 리스트 색상으로 채워지는 원형 체크 |

### 3.3 스마트 리스트 카드 색상 (Apple 기본값)

| 스마트 리스트 | 배경색 | 아이콘 |
|--------------|--------|--------|
| 오늘 | `#007AFF` (파랑) | 캘린더 오늘 날짜 |
| 예정 | `#FF3B30` (빨강) | 캘린더 |
| 전체 | `#1C1C1E` (검정) | 트레이 |
| 플래그 지정됨 | `#FF9500` (주황) | 깃발 |
| 완료됨 | `#8E8E93` (회색) | 체크마크 |

### 3.4 리스트 색상 프리셋 (12가지)

```
빨강 #FF3B30 | 주황 #FF9500 | 노랑 #FFCC00 | 초록 #34C759
민트 #00C7BE | 시안 #32ADE6 | 파랑 #007AFF | 남색 #5856D6
보라 #AF52DE | 핑크 #FF2D55 | 브라운 #A2845E | 회색 #8E8E93
```

### 3.5 전체 레이아웃

```
┌──────────────────────────────────────────────────────────────┐

│ 🔍 검색 바                     │                              │

│ ┌────────────┬────────────┐   │  리스트 제목 (큰 볼드, 리스트  │
│ │ 📅 오늘    │ 📆 예정    │   │  색상 적용)                   │
│ │    3       │    5       │   │  리마인더 수                   │
│ ├────────────┼────────────┤   │  ────────────────────────── │
│ │ 📋 전체    │ 🚩 플래그  │   │                              │
│ │   12       │    2       │   │  ○ 리마인더 제목              │
│ ├────────────┴────────────┤   │    메모 미리보기 · 내일        │
│ │ ✅ 완료됨               │   │                              │
│ └─────────────────────────┘   │  ○ 리마인더 제목              │
│                                │    우선순위 !!! · 오늘         │
│ 내 리스트                      │                              │
│ ─────────────────────────     │  ● 완료된 리마인더 ~~취소선~~  │
│ 🔴 개인                    3  │                              │
│ 🔵 업무                    5  │                              │
│ 🟢 쇼핑                    2  │                              │
│                                │                              │
│                                │                              │
│ [+ 리스트 추가]                │  [+ 새로운 리마인더]           │

```

### 3.6 스마트 리스트 카드 (2x2 그리드 + 1)

- 사이드바 상단에 2x2 그리드로 오늘/예정/전체/플래그 표시
- 각 카드: 둥근 모서리(12px), 아이콘(원형 배경) + 카운트(우측 상단, 볼드)
- 완료됨은 그리드 아래 전체 너비로 표시
- 카드 hover 시 약간의 배경색 변화

### 3.7 사이드바 리스트 항목

- 왼쪽: 리스트 색상의 원형 아이콘 (채워진 원 안에 아이콘)
- 가운데: 리스트 이름
- 오른쪽: 미완료 리마인더 수 (회색 텍스트)
- 선택 시: 배경 하이라이트
- 우클릭 시: 컨텍스트 메뉴 (수정, 삭제)

### 3.8 메인 콘텐츠 - 리마인더 목록

- **헤더**: 리스트 이름 (24px 볼드, 리스트 색상 적용) + 리마인더 수
- **리마인더 행**:
  - 왼쪽: 원형 체크박스 (미완료: 빈 원, 테두리는 리스트 색상)
  - 체크 시: 원이 리스트 색상으로 채워지며 체크마크 표시 + 제목에 취소선 + 페이드아웃 애니메이션
  - 제목 아래: 보조 정보 (메모 미리보기, 마감일, 우선순위 표시)
  - 우측: 플래그 아이콘 (설정 시 주황색), 상세보기 info 버튼
- **구분선**: 각 행 사이 얇은 구분선 (왼쪽 체크박스 이후부터 시작, Apple 스타일)

### 3.9 리마인더 상세 (인라인 확장)

리마인더 클릭 시 해당 행이 확장되며 상세 편집 영역이 나타남 (Apple 스타일 인라인 편집):

```
┌─────────────────────────────────────────┐
│ ○  리마인더 제목 (인라인 편집)            │
│    ┌─────────────────────────────────┐  │
│    │ 메모 입력 영역                    │  │
│    └─────────────────────────────────┘  │
│    ┌─────────────────────────────────┐  │
│    │ 📅 날짜     2026-03-20          │  │
│    │ ⏰ 시간     오전 9:00            │  │
│    │ 🚩 플래그    ○                   │  │
│    │ ❗ 우선순위  보통 ▼               │  │
│    │ 📋 리스트   업무 ▼               │  │
│    └─────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

- 상세 영역은 둥근 모서리 카드 형태
- 각 필드는 행 형태로 레이블 + 값 표시
- 값 클릭 시 편집 가능 (인라인 에디팅)

### 3.10 새 리마인더 추가

- 하단 `+ 새로운 리마인더` 클릭 시 목록 맨 아래에 빈 행이 추가됨
- 커서가 제목 입력에 자동 포커스
- Enter 키로 빠른 생성 후 다음 항목 연속 입력 가능

### 3.11 애니메이션 & 인터랙션

| 인터랙션 | 애니메이션 |
|---------|----------|
| 완료 체크 | 원형 체크박스가 색상으로 채워지며 체크마크 등장 (0.3s) |
| 완료 후 사라짐 | 0.5s 딜레이 후 높이가 줄어들며 페이드아웃 |
| 리스트 전환 | 메인 콘텐츠 영역 크로스페이드 (0.2s) |
| 상세 확장 | 높이 확장 애니메이션 (0.25s ease) |
| 사이드바 hover | 배경색 변화 (0.15s) |
| 리마인더 삭제 | 좌측으로 스와이프 또는 우클릭 삭제 시 슬라이드아웃 |

## 4. API 설계

### 4.1 리스트 API

| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | /api/lists | 전체 리스트 조회 |
| POST | /api/lists | 리스트 생성 |
| PUT | /api/lists/{id} | 리스트 수정 |
| DELETE | /api/lists/{id} | 리스트 삭제 |
| PUT | /api/lists/reorder | 리스트 순서 변경 |

### 4.2 리마인더 API

| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | /api/reminders?listId={id} | 리스트별 리마인더 조회 |
| GET | /api/reminders/today | 오늘 마감 리마인더 |
| GET | /api/reminders/scheduled | 예정된 리마인더 |
| GET | /api/reminders/all | 전체 미완료 리마인더 |
| GET | /api/reminders/flagged | 플래그 리마인더 |
| GET | /api/reminders/completed | 완료된 리마인더 |
| GET | /api/reminders/search?q={keyword} | 검색 |
| POST | /api/reminders | 리마인더 생성 |
| PUT | /api/reminders/{id} | 리마인더 수정 |
| PATCH | /api/reminders/{id}/toggle | 완료 토글 |
| DELETE | /api/reminders/{id} | 리마인더 삭제 |
